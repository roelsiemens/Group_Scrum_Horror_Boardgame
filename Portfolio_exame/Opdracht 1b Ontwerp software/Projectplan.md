# Projectplan

## Afspraken

- Als je ziek bent geef het vroegtijdig aan via nexchat

- Voor elke merge je werk laten nakijken door je teamleden

- Elke ochtend een korte daily standup doen

- Als een functie te moelijk is of er een verandering in de planning moet zijn dat dan aangeven aan je team.

- tijdens het nakijken van een pull request moet de maker van de pull request aanwezig zijn

- na het nakijken duidelijke aangeven wat er goed en fout was aan de pull request via een comment

- aan het einde van de dag een korte reflectie gesprek houden


## Definition of done

Een issue is klaar wanneer:

- De acceptatiecriteria van de issue zijn behaald.
- Er is een pull request gemaakt voor de issue.
- De pull request is nagekeken door een teamgenoot.
- De code is geaccepteerd.
- De pull request is naar de merge branch gemerged door degene die de pull request
gemaakt heeft
